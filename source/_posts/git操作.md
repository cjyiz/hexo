### 一.检查文件及仓库状态
`git status`:每次租了改动并执行了`git add`之后，可以执行`git status`指令，用于检查当前文件状态，查看文件改动。执行该指令后会提示你一些相关操作指令。

### 二.回顾提交历史
`git log`:在提交了若干更新之后，又克隆了某个项目，想回顾下提交历史，可以使用该指令查看。

### 三.撤销操作
取消了已经提交到缓存区文件:`git reset HEAD 文件名.后缀`
取消本次文件内容的改动:`git checkout -- 文件名.后缀`(谨慎使用)

### 四.远程仓库及分支的使用
`git remote`:用于查看当前配置有哪些远程仓库分支，他会列出每个远程分支的简单名字。在克隆完整个项目，至少可以看到一个名为`origin`的远程库。
`git remoter -v`:显示远程分支对应的clone地址。
`git remote add 分支本地名 [url]`：执行完毕后就可以用本地分支名取代远程仓库地址。
`git fetch 分支本地名`：紧接上一条操作，改名了可以将远程分支数据抓取到本地，但是不会自动合并到当前工作分支，需要手动合并。
`git push 本地分支名 远程分支名`:将本地分支推送到远程分支上。
`git remote rm 本地分支名`:移除本地分支。

### 五.分支创建和合并
`git branch`:给出当前所有分支清单。
`git branch 分支名`:新建一个分支
`git checkout 分支名`:切换到新建分支名下
`git checkout -b 分支名`：创建一个分支，并切换到该分支下。
`git merge 分支名2`:将分支名2合并到当前操作的分支上。一般是`master`分支。
`git branch -d 分支名`:删除无用分支
